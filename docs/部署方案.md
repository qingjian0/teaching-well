# 乡村教师教学助手部署方案

## 1. 部署概述

乡村教师教学助手是一个纯前端项目，无需后端服务，部署相对简单。本方案提供两种部署方式：本地部署和服务器部署。

## 2. 部署环境要求

### 2.1 本地部署

- 现代浏览器（Chrome、Firefox、Safari、Edge等）
- 操作系统：Windows、macOS、Linux均可

### 2.2 服务器部署

- Web服务器（Apache、Nginx、IIS等）
- 操作系统：Linux、Windows Server均可
- 网络连接：稳定的互联网连接

## 3. 部署步骤

### 3.1 本地部署

1. **获取项目文件**：
   - 下载项目压缩包并解压，或克隆项目仓库
   - 确保项目目录结构完整

2. **运行项目**：
   - 直接双击 `index.html` 文件
   - 系统会在默认浏览器中打开

3. **验证部署**：
   - 检查首页是否正常加载
   - 测试各功能模块是否正常运行

### 3.2 服务器部署

1. **准备服务器**：
   - 确保Web服务器已安装并运行
   - 配置服务器的文档根目录

2. **上传项目文件**：
   - 使用FTP、SCP等工具将项目文件上传到服务器
   - 确保文件权限设置正确

3. **配置服务器**：
   - 对于Apache服务器，确保 `httpd.conf` 或 `.htaccess` 配置正确
   - 对于Nginx服务器，确保 `nginx.conf` 配置正确
   - 确保服务器支持静态文件访问

4. **测试部署**：
   - 通过服务器地址访问系统
   - 检查各功能模块是否正常运行

## 4. 部署配置

### 4.1 本地部署配置

本地部署无需特殊配置，直接打开 `index.html` 文件即可。

### 4.2 服务器部署配置

#### 4.2.1 Apache服务器配置

```apache
<VirtualHost *:80>
    ServerName teaching-well.example.com
    DocumentRoot /var/www/teaching-well
    
    <Directory /var/www/teaching-well>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
```

#### 4.2.2 Nginx服务器配置

```nginx
server {
    listen 80;
    server_name teaching-well.example.com;
    
    root /var/www/teaching-well;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
```

## 5. 性能优化

### 5.1 资源压缩

- 压缩CSS和JavaScript文件
- 优化图片资源
- 使用CDN加速静态资源

### 5.2 缓存策略

- 设置适当的缓存头
- 启用浏览器缓存
- 使用ETag或Last-Modified头

### 5.3 加载优化

- 延迟加载非关键资源
- 优化字体加载
- 减少HTTP请求

## 6. 安全配置

### 6.1 服务器安全

- 定期更新服务器软件
- 配置防火墙规则
- 限制不必要的端口访问

### 6.2 内容安全

- 配置Content-Security-Policy头
- 防止XSS攻击
- 保护本地存储数据

## 8. 故障排查

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 页面无法加载 | 服务器配置错误 | 检查服务器配置 |
| 功能无响应 | AI能力未初始化 | 检查网络连接，确保TRAE SDK正常加载 |
| 资源无法上传 | 服务器权限不足 | 检查文件权限设置 |
| 页面加载缓慢 | 资源过大 | 优化资源大小，启用压缩 |

### 8.2 排查步骤

1. 检查服务器日志
2. 检查浏览器控制台错误
3. 验证网络连接
4. 检查文件权限
5. 测试基本功能
