# 教案活井 - 项目部署与参赛提交指南

## 一、项目部署到互联网

### 方法1：部署到GitHub Pages（推荐，免费）

#### 步骤1：创建GitHub仓库
1. 访问 https://github.com 并登录
2. 点击右上角"+" → "New repository"
3. 仓库名称填写：`teaching-well`
4. 选择"Public"（公开）
5. 点击"Create repository"

#### 步骤2：上传项目文件
1. 在新建的仓库页面，点击"uploading an existing file"
2. 将 `/workspace/teaching-well/` 目录下的所有文件拖拽上传
3. 点击"Commit changes"

#### 步骤3：启用GitHub Pages
1. 进入仓库的"Settings"（设置）选项卡
2. 滚动到"GitHub Pages"部分
3. Source（源）选择："Deploy from a branch"
4. Branch（分支）选择："main"，文件夹选择："/ (root)"
5. 点击"Save"
6. 等待1-2分钟，页面会显示你的网站地址，格式为：`https://你的用户名.github.io/teaching-well/`

### 方法2：部署到Vercel（推荐，快速）

#### 步骤1：注册Vercel账号
1. 访问 https://vercel.com 并使用GitHub账号登录
2. 点击"Add New" → "Project"
3. 选择"Import Git Repository"（导入Git仓库）
4. 选择刚创建的GitHub仓库
5. Framework Preset保持默认
6. 点击"Deploy"

#### 步骤2：获取访问地址
部署完成后，Vercel会提供一个访问地址，格式为：`https://teaching-well.vercel.app/` 或类似地址

### 方法3：部署到Netlify（备选）

#### 步骤1：注册Netlify账号
1. 访问 https://netlify.com 并使用GitHub账号登录
2. 点击"Add new site" → "Import an existing project"
3. 选择"Deploy manually"（手动部署）
4. 将 `/workspace/teaching-well/` 目录下的所有文件拖拽上传
5. 等待部署完成，获取访问地址

### 方法4：使用Git命令上传（适用于有Git经验的用户）

```bash
# 1. 进入项目目录
cd /workspace/teaching-well

# 2. 初始化Git仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "Initial commit: 教案活井项目"

# 5. 添加远程仓库（将 YOUR_USERNAME 替换为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/teaching-well.git

# 6. 推送到GitHub
git push -u origin main
```

##### ③ 作品介绍（必须）
- **作品名称**：教案活井
- **作品类型**：网页工具
- **核心功能**：
  - 教案模板库（50+套预设模板）
  - 一键微调生成
  - 教案一键分享
  - 教材版本适配
  - 弱网离线模式
  - 内容纠错反馈
  - 教研群共享池
  - 专家审核标记

##### ④ 用 SOLO 实现的过程（必须）
简述你如何使用TRAE SOLO完成任务：
1. 通过TRAE SOLO进行项目规划和设计
2. 使用TRAE SOLO生成代码框架
3. 利用TRAE SOLO优化和完善功能
4. 通过TRAE SOLO进行测试和调试

##### ⑤ 成果展示（必须）
